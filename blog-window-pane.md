# tmux 会话管理只是开始——tmuxtui 的窗口与分屏才是效率杀手锏

上一篇介绍了 tmuxtui 的会话管理——浏览、创建、接入、销毁，解决了「tmux 命令记不住」的痛点。但会话只是第一层，tmux 真正的效率来源是**窗口**和**分屏（Pane）**。

这一篇，我们直接进入 tmuxtui 的窗口管理和 Pane 布局功能。

## 前置准备

还没装？一行命令：

```bash
npm install -g tmux-tui
```

启动：

```bash
tmuxtui
```

会话的基本操作回顾：`↑↓` 选择，`Enter` 接入，`n` 新建，`r` 重命名，`d` 销毁，`q` 退出。

## 窗口管理：按一个 `c` 就够了

在会话列表中，选中任意会话，按 `c`，进入**配置模式**。

配置模式是专门管理某个会话下所有窗口的界面。你会看到该会话里的窗口列表，以及以下操作：

| 按键 | 操作 |
|------|------|
| `↑` `↓` | 选择窗口 |
| `n` | 新建窗口 |
| `r` | 重命名窗口 |
| `d` | 删除窗口（有确认提示） |
| `i` | 初始化 Pane 布局 |
| `Esc` | 返回会话列表 |

操作逻辑和会话管理完全一致——上手零成本。

实际场景举例：

- 你有一个 `my-project` 会话，想加一个窗口跑测试？选中它，按 `c`，再按 `n`，输入名称 `test`，回车。
- 想给窗口取个更有意义的名字？选中窗口按 `r`，改成 `server`、`logs`、`tests`，一目了然。
- 某个窗口不再需要了？选中按 `d`，确认后删除。

## Pane 布局：9 种预设，按 `i` 直接选

在配置模式中，选中一个**单 Pane 且非活跃**的窗口，按 `i`，你会看到 9 种内置布局：

| 布局 | Pane 数 | 说明 |
|------|---------|------|
| 1. Single | 1 | 不分屏，保持原样 |
| 2. H-Split | 2 | 左右对半分 |
| 3. V-Split | 2 | 上下对半分 |
| 4. IDE | 2 | 左侧窄栏（25%）+ 右侧主区域 |
| 5. IDE+Term | 3 | IDE 布局 + 底部终端 |
| 6. Triple | 3 | 三列等分 |
| 7. IDE+Side | 3 | IDE + 右侧边栏 |
| 8. Quad | 4 | 2×2 四宫格 |
| 9. IDE Pro | 4 | IDE + 右侧上下分屏 |

选择时有实时预览，你能直接看到分屏效果，选中满意的按回车确认。

### 几个典型用法

**写代码 + 跑命令**：选布局 4（IDE），左侧放文件树或导航，右侧写代码。

**全功能开发环境**：选布局 5（IDE+Term），上面写代码，下面跑测试/服务器，一个窗口搞定。

**同时看多个日志**：选布局 8（Quad），四个 Pane 分别 tail 不同的日志文件。

**前后端分离**：选布局 6（Triple），三列分别跑前端、后端、数据库。

## 操作流程总结

一条链路串起来：

```
tmuxtui → 选中会话 → 按 c → 选中窗口 → 按 i → 选布局 → 回车确认
```

如果你想给一个新项目快速搭建工作环境：

```bash
# 1. 用当前目录创建会话
tmuxtui init

# 2. 进入后按 c 配置窗口
# 3. 按 n 新建窗口，按 i 选择布局
# 4. 搞定，开始写代码
```

整个过程不需要记任何 tmux 命令。
